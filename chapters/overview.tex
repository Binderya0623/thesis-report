Энэ бүлэгт судалгааны ажлын үндэслэл, хүрэх зорилго болон зорилтуудыг танилцуулна.
\section{Сэдэв сонгосон үндэслэл}
Вэб аппликейшн хөгжүүлэлтэд орчин үеийн JavaScript фрэймворкууд өргөн ашиглагдаж байгаа хэдий ч, зарим тохиолдолд серверийн талд рендер хийгддэг уламжлалт технологитой хослуулан ашиглах шаардлага гардаг. Ийм ялгаатай технологийн парадигмуудыг нэгтгэж нэгдсэн хэрэглэгчийн интерфейс үүсгэх нь архитектурын хувьд хүндрэлтэй байдаг. Иймд шинээр хөгжүүлэгдэж буй МУИС-ийн Дипломын ажлын удирдлагын системийн хөгжүүлэлтийн нэгэн хувилбар болгон холимог микрофронтэнд архитектурыг сонгон авч, React, JSF фрэйм- воркуудыг хэрхэн уялдуулан ажиллуулж болохыг бодитоор хэрэгжүүлэн, туршилт хийж судлах нь энэхүү сэдвийг сонгох үндэслэл болсон болно.
\section{Зорилго}
	\quad \quad	МУИС-ийн Дипломын ажлын удирдлагын системийн хэрэглэгчийн интерфэйс дээр React болон JSF технологийг хослуулсан микрофронтэнд архитектурыг туршина.
\section{Зорилт}
	Дээрх зорилгод хүрэхийн тулд дараах зорилтуудыг дэвшүүлэн ажиллана. Үүнд:
\begin{enumerate}
	\item Судалгаа хийх зорилтоор микрофронтэнд архитектурын онол, арга зүйг судлах;
	\item Туршилтын загвар боловсруулах зорилтоор React, JSF ашиглан холимог архитектур бүхий системийн туршилтын загварыг хөгжүүлэх;
	\item Туршилт ба үнэлгээ хийх зорилтоор боловсруулсан загвар дээрээ туршилт хийж, ISO/IEC 25010 стандарт болон DEMATEL аргачлалаар үнэлж, дүгнэнэ.
\end{enumerate}

\section{Сэдвийн судлагдсан байдал}
Энэхүү судалгааны ажлын хүрээнд Scopus болон DiVA өгөгдлийн сангаас 5 бүтээлийг сонгон шинжиллээ. Эдгээр судалгаанууд нь голчлон чанарыг ISO 25010 стандартаар үнэлсэн нь энэ судалгааны арга зүйтэй нийцнэ. Дээрх судалгаануудын гол үр дүнг хураангуйлбал:

\begin{itemize}
    \item S. Mohammed нар (2022): Эмнэлгийн чатбот систем дээр микрофронтэнд туршиж,хэрэг- жүүлэлт нь тухайн системийн тохиргооноос ихээхэн хамаардаг болохыг харуулсан.
    \item E. Stefanovska нар (2022): Module Federation ашиглан микрофронтэндүүдийг статик файл хэлбэрээр татах нь системийг үйлдвэрлэлд нэвтрүүлэх хурдыг эрс нэмэгдүүлдэг болохыг баталсан.
    \item A. Pavlenko нар (2020): Микрофронтэнд нь том төсөлд маш тохиромжтой боловч, жижиг багийн хувьд бүтцийг хэт төвөгтэй болгож, өндөр туршлага шаарддаг болохыг тогтоосон.
    \item P. Y. Tilak нар (2020): Модулиудыг зэрэгцээ хөгжүүлэх, хэсэгчлэн шинэчлэх боломж бүрдсэнээр хөгжүүлэгчийн бүтээмж, нөөцийн зарцуулалт илүү сайжирдгийг онцолсон.
    \item A. Montelius нар (2021): Туршилтын жижиг загвараас гарсан үр дүнг бүх системд шууд хамааруулах боломжгүй бөгөөд микрофронтэндийн үр ашиг төслийн цар хүрээнээс шууд хамаарна гэж дүгнэсэн.
\end{itemize}

\section{Судалгааны ажлын шинэлэг тал}
Энэ бакалаврын судалгааны ажлын шинэлэг тал нь дараах хоёр хэсгээс бүрдэнэ. Үүнд:
\begin{enumerate}
    \item Серверийн талын JSF болон клиент талын React фрэймворкийг \texttt{<script>} интеграци ашиглан DOM түвшинд нэгтгэсэн туршилтын загвар хөгжүүлсэн.
    \item Хэрэгжүүлсэн холимог микрофронтэнд архитектурыг ISO/IEC 25010 чанарын стандарт болон DEMATEL математик аргачлалыг ашиглан системийн чанарын үзүүлэлтүүд хоорондын шалтгаан-үр дагаварын хамаарлыг үнэлж дүгнэсэн.
\end{enumerate}
\section{Судалгааны ажлын үр дүн}
\textbf{Үр дүн:}
Энэ судалгааны хүрээнд JSF, React фрэймворкуудыг хослуулсан микрофронтэнд архитектурын туршилтын загварыг хэрэгжүүлж, модулиудыг бие даан ажиллуулж, интеграци хийх боломжтойг харуулна. Мөн DEMATEL аргачлалын дагуу чанарын шалгуур үзүүлэл- түүдийн хамаарлыг тодорхойлж, үнэлгээний үр дүнг гаргана.

\textbf{Ач холбогдол:}
Энэ судалгаа нь өөр өөр технологийн стэк бүхий аппликэйшнүүдийг микрофронтэнд архитектураар хэрхэн нэгтгэх техникийн шийдлийг бодитоор харуулсан кейс судалгаа болно. Цаашид ижил төстэй технологийн хоршил ашиглах хэрэгцээ гарсан оюутнуу- дад техникийн лавлагаа, харьцуулалт хийх суурь мэдээлэл болох ач холбогдолтой юм.
