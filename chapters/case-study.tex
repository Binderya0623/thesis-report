\section{Турших загвар}
\subsection{Турших загварын бүтэц}
Туршилтын үе шатанд ашигласан техникийн мэдлэг болон туршлагыг МУИС-ийн Дипломын ажлыг удирдах системийн холимог микрофронтэнд загварыг хөгжүүлэх явцад олж авсан болно. Энэхүү загвар нь уламжлалт JSF технологийг хост болгон ашиглаж, орчин үеийн React микрофронтэндүүдийг нэгтгэх архитектурыг турших зорилготой.

\subsection{Турших загварын бүтэц}
Турших загварын бүтцийг Зураг \ref{fig:prototype-structure}-д тодорхойлсон. Энэ бүтэц нь дараах үндсэн бүрэлдэхүүн хэсгүүдээс бүрдэнэ:

\begin{enumerate}
    \item \textbf{Үндсэн аппликэйшн:} Энэ нь хэрэглэгчид харагдах үндсэн бүрхүүл бөгөөд дотроо React аппликэйшнүүд ачааллах зориулалттай хоосон DOM элементүүдийг (жишээ нь \texttt{<div id="microfrontend-root"></div>}) бэлтгэж өгнө.
    \item \textbf{React Микрофронтэнд модулиуд:} Эдгээр нь тус бүрдээ бие даасан хавтас болон тохиргоотой байна. зурагт үзүүлсэнтэй ижил зарчмаар модуль тус бүр \texttt{package.json} болон \texttt{webpack.config.js} файлуудтай байх бөгөөд бие даан хөгжүүлэгдэж, хөрвүүлэгдэнэ.
\end{enumerate}

\subsection{Webpack тохиргоо ба Интеграци}
Орчин үеийн JavaScript фрэймворкууд нь Webpack хэрэгсэл дээр суурилдаг бөгөөд төслийг ажиллуулах скриптүүдийг \texttt{package.json} дотор тохируулдаг. зурагт үзүүлсний дагуу \texttt{"start"} болон \texttt{"build"} скриптүүдийг ашиглан микрофронтэнд модулиудыг хөгжүүлэлтийн орчинд бие даан ажиллуулна.

Харин эдгээр React модулиудыг JSF хосттой нэгтгэхдээ Webpack Module Federation эсвэл шууд багцлагдсан JavaScript файлыг нь ашиглана. JSF-ийн \texttt{.xhtml} хуудсан дотор Webpack-аар үүсгэгдсэн \texttt{remoteEntry.js} эсвэл \texttt{main.js} файлыг энгийн \texttt{<script>} таг ашиглан дуудна.

Зураг \ref{fig:package-update}-д харуулснаар, \texttt{package.json} файл дахь скриптийг тохируулснаар хөгжүүлэлтийн орчинд Webpack-ийг ажиллуулах боломжтой болно.

\subsection{Асинхрон скрипт ачаалалт}
Микрофронтэндүүдийг ачааллах үед шаардлагатай депенденси бүрэн татагдаж дуусаагүй байхад аппликэйшн уншигдаж эхэлбэл алдаа гарах эрсдэлтэй. Үүнээс сэргийлэхийн тулд асинхрон скрипт ачаалалтыг ашигладаг. Зураг \ref{fig:async}-т харуулсанчлан, аппликэйшнийг эхлүүлэх логикийг \texttt{bootstrap.js} файл дотор бичиж, \texttt{index.js} нь зөвхөн \texttt{import('./bootstrap');} гэсэн динамик дуудлагыг агуулна. Энэ нь хөтчид модулиудын хамаарлыг урьдчилан татах боломжийг олгож, системийн найдвартай ажиллагааг хангана.

\subsection{Загварын давхцал (CSS issue)}
Зураг \ref{fig:css-issue}-т дүрсэлснээр, хэрэглэгч нэг микрофронтэндийн хуудаснаас нөгөө рүү шилжих үед хөтөч нь тухайн модулиудын CSS загваруудыг DOM-ийн \texttt{<head>} хэсэгт нэмж ачаалдаг бөгөөд ижил нэртэй CSS классууд нэгнийгээ дарж бичин, UI-ийн хэв гажилт үүсгэдэг байна. Энэ асуудлаас сэргийлэхийн тулд туршилтын загварт React модулиудын хөгжүүлэлтэдCSS Modules эсвэл Scoped CSS аргачлалыг ашиглах шаардлагатай. Энэ нь Webpack хөрвүүлэлтийн явцад CSS класс бүрийн нэрэнд дахин давтагдахгүй хэш утга залгаж өгснөөр JSF үндсэн аппликэйшн болон бусад React модулиудын загвартай мөргөлдөхөөс бүрэн хамгаална.
